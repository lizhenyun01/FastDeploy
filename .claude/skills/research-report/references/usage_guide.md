# Research Report Skill — 使用指南与技巧

## 快速开始

### 最简单的用法

直接告诉我你想研究什么：

```
帮我研究一下 Flash Attention v2 和 v3 的区别
```

```
调研 MLA（多头潜在注意力）的 KV Cache 压缩原理，生成一份 HTML 报告
```

```
Research CUDA Tensor Core programming, make a visual report
```

我会自动：
1. 搜集资料并整合知识
2. 生成一份美观的 HTML 报告
3. 保存到你指定的位置（或询问你）
4. 更新研究索引页面

---

## 提示词技巧

### 🎯 指定报告深度

| 提示词写法 | 效果 |
|---|---|
| "简单介绍一下 X" | 概述型，偏广度，4-5 节 |
| "深入调研 X" | 深度型，含原理推导，6-8 节 |
| "X 和 Y 的对比分析" | 对比型，含对比表格和结论建议 |
| "X 的工程实践指南" | 实践型，含代码示例和踩坑记录 |

### 📂 指定保存位置

```
帮我研究 Flash Decoding，保存到 ~/Research/GPU优化/
```

```
调研 RoPE 外推方案，保存在当前目录下
```

### 🏷️ 指定标签（影响索引分类）

```
研究 CUDA Swizzle 原理，标签：CUDA, 内存优化, SM80
```

### 📊 请求特定可视化元素

```
帮我研究 PagedAttention，要包含内存管理的架构图
```

```
对比 vLLM、TensorRT-LLM 和 SGLang，要做一个详细的性能对比表
```

```
调研 Speculative Decoding，需要包含流程图说明每个步骤
```

## 报告结构说明

每份报告包含以下模块（根据主题自动调整）：

| 模块 | 说明 |
|---|---|
| **封面** | 标题、日期、技术标签、一句话摘要 |
| **TL;DR** | 3-5 条核心要点，30 秒读完 |
| **背景** | 为什么这个技术存在，解决什么问题 |
| **核心知识点** | 技术原理、数学推导、架构设计 |
| **对比分析** | 与其他方案的横向对比（有时适用） |
| **结论与建议** | 实际应用建议、选型指导 |
| **参考资料** | 论文、博客、代码链接 |

---

## 索引页功能

生成的 `index.html` 索引页支持：

- **🔍 搜索**：按标题或标签关键词实时过滤
- **🏷️ 标签筛选**：点击标签按类别筛选报告
- **📅 时间排序**：报告按日期倒序排列（最新在前）
- **📊 统计信息**：报告总数、标签数量、最近更新日期

---

## 为报告添加元数据（高级）

如果你手动创建 HTML 报告，在 `<head>` 中添加以下注释，让索引能正确识别：

```html
<!-- RESEARCH_META
title: 我的研究报告标题
date: 2026-04-09
tags: CUDA, Attention, GPU
summary: 这里是一句话摘要，会显示在索引卡片上。
-->
```

## 与 AIbin-Note 的关系

如果你已经部署了 [AIbin-Note](https://github.com/chang-wenbin/AIbin-Note)（GitHub Pages 网站：https://chang-wenbin.github.io/AIbin-Note/），
可以把生成的研究报告复制到对应分类目录中，然后运行：

```bash
python3 AIbin-Note/update_index.py --push
```

这样研究报告就会自动出现在你的 GitHub Pages 网站上。

## 常见问题

**Q：报告可以离线查看吗？**
A：可以，所有 CSS 都内嵌在 HTML 中，唯一的外部依赖是 Mermaid（架构图库）。
如果没有网络，图表不会显示，但其他内容完全正常。

**Q：如何让报告支持中英文双语？**
A：在提示词中说明即可：`"生成中英双语对照的研究报告"`

**Q：报告太长/太短怎么办？**
A：在提示词中明确：`"要非常详细"` / `"简洁版，控制在 3 个核心要点"`

**Q：能批量生成多份报告吗？**
A：可以，把多个研究问题列成清单发给我，我会逐一生成。
